# Sevent — Saudi Compliance & Required Documents

> Prepared for launch in the Kingdom of Saudi Arabia. Last updated: May 2026.

---

## 1. Documents Required to Use Sevent

### 1.1 Supplier Onboarding Documents

Suppliers must upload the following during registration. Documents are reviewed by Sevent's admin team before the supplier profile is published.

| Document | Who Needs It | Format | Notes |
|----------|-------------|--------|-------|
| **Commercial Registration (CR) Certificate** | Companies | PDF, ≤10 MB | السجل التجاري. Issued by Ministry of Commerce. Proves legal business existence. |
| **VAT Registration Certificate** | Companies (VAT-registered) | PDF, ≤10 MB | شهادة التسجيل في ضريبة القيمة المضافة. Issued by ZATCA. |
| **National Address Certificate** | Companies | PDF, ≤10 MB | شهادة العنوان الوطني. Issued by Saudi Post (SPL). Required for e-invoicing by ZATCA. |
| **IBAN Certificate** | All suppliers | PDF, ≤10 MB | شهادة الآيبان. Issued by the supplier's bank. Required for off-platform bank transfers. |
| **National ID** | Freelancers | String (ID number) | رقم الهوية الوطنية. 10 digits. For sole proprietors / freelancers. |
| **Company Profile** | All suppliers (optional) | PDF, ≤10 MB | Portfolio or capability statement for admin review. |
| **Logo** | All suppliers | Image, ≤1 MB | Displayed on public supplier profile. |
| **GEA Permit** | Entertainment suppliers | PDF, ≤10 MB | تصريح الهيئة العامة للترفيه. Required for entertainment categories (performers, DJs, etc.). |
| **Professional Certifications** | Optional | PDF, ≤10 MB | Any industry-specific certifications the supplier wants verified. |
| **Other Documents** | As needed | PDF, ≤10 MB | Catch-all for additional verification documents. |

**Legal types and document requirements:**

| Legal Type | Minimum Required Documents |
|------------|---------------------------|
| **Company (شركة)** | CR certificate, VAT certificate, National Address certificate, IBAN certificate, Logo |
| **Freelancer (مستقل)** | National ID, IBAN certificate, Logo |
| **Foreign (أجنبي)** | Equivalent foreign registration documents, IBAN certificate, Logo |

### 1.2 Organizer (Event Creator) Requirements

Organizers go through a lighter registration flow:

| Requirement | Details |
|-------------|---------|
| **Full name** | As it appears on legal documents |
| **Email address** | Verified via magic link / OTP |
| **Phone number** | Saudi mobile (05xxxxxxxx, 9 digits) |
| **Company name** (if corporate) | Optional — for corporate/government organizers |
| **No document uploads** required in v1 | Organizers are verified by their payment method (off-platform bank transfer in pilot) |

### 1.3 Documents Generated by the Platform

Sevent generates the following legal/regulatory documents:

| Document | When | Purpose |
|----------|------|---------|
| **Quote Revision (PDF)** | When a supplier submits a quote | Immutable, SHA-256 hashed snapshot of pricing and terms. Accepted revision becomes the binding agreement. |
| **Booking Contract (PDF)** | After supplier confirms booking | Renders the accepted quote revision into a formal contract. |
| **Payment Schedule** | Embedded in booking contract | Deposit (25%), balance (50%), final release (25%) breakdown. |
| **Dispute Record** | If a dispute is opened | Evidence files + notes per party, resolution outcome. |

## 2. Regulatory Compliance Landscape

### 2.1 SAMA — Payment Service Provider Regulation

**Authority:** Saudi Central Bank (البنك المركزي السعودي)

| Item | Status / Action |
|------|-----------------|
| **What applies** | If Sevent holds customer funds (escrow), it must hold a PSP license from SAMA. |
| **Pilot stance (v1.0)** | Off-platform bank transfers. No customer funds held. **No SAMA license needed.** |
| **Post-pilot (v1.x)** | Partner with **Tap Payments** (SAMA-licensed). Escrow funds sit in Tap's infrastructure, not Sevent's. This avoids the need for a separate PSP license. |
| **PSP license thresholds** | Micro PI: SAR 1M minimum paid-up capital. Major PI: SAR 50M+. |
| **AML/KYC obligations** | Required regardless. Implement supplier identity verification (CR check, National ID) and transaction monitoring. Keep AML records for 10 years. |
| **Customer fund segregation** | If Sevent ever holds funds directly, they must be in segregated accounts, not operating accounts. |

### 2.2 ZATCA — VAT & E-Invoicing (FATOORAH)

**Authority:** Zakat, Tax and Customs Authority (هيئة الزكاة والضريبة والجمارك)

| Item | Status / Action |
|------|-----------------|
| **VAT rate** | 15% standard rate (implemented in pricing engine: `src/lib/domain/pricing/engine.ts:54`) |
| **Deemed supplier rule** | Article 47(3): An electronic marketplace controlling transaction terms is deemed the supplier. Must collect and remit 15% VAT on the **full transaction value**, not just commission. |
| **Pilot stance (v1.0)** | Off-platform bank transfers. Sevent is NOT processing payments, so deemed-supplier rule is **not yet triggered**. VAT is displayed as a line item but collection/remittance is the supplier's responsibility. |
| **Post-pilot (v1.x)** | Must integrate ZATCA FATOORAH e-invoicing before processing payments. |
| **FATOORAH requirements** | UBL 2.1 XML format, Arabic-language invoices, cryptographic stamps, QR codes. Real-time clearance for B2B invoices, 24-hour reporting for B2C. |
| **Penalties** | SAR 5,000–50,000 per violation for non-compliant invoices. |
| **National Address** | Required in e-invoices. Collect supplier national address certificates at onboarding (already wired). |

### 2.3 PDPL — Personal Data Protection Law

**Authority:** Saudi Data & AI Authority (SDAIA) / National Data Management Office (NDMO)

| Item | Status / Action |
|------|-----------------|
| **Effective** | Enforced since September 2024. |
| **Scope** | Covers all personal data processing of individuals in KSA, regardless of where the processor is located. |
| **Data collected by Sevent** | Full names, emails, phone numbers, National ID numbers, CR numbers, business details, event locations, bank account details (IBAN), verification documents. |
| **Required measures** | |
| | — **Consent**: Explicit opt-in for data collection. Privacy policy must be accepted at registration. |
| | — **Purpose limitation**: Data used only for stated purposes (verification, matching, payments). |
| | — **Data minimization**: Only collect what's necessary. ✅ Already practiced. |
| | — **Retention policy**: Stated in privacy policy. Verification docs + financial records retained per Saudi commercial law. Account data deleted 6 months after closure. |
| | — **User rights**: Access, correct, delete personal data within 30 days (stated in privacy policy). |
| | — **Data breach notification**: Must notify SDAIA within 72 hours of breach discovery. |
| | — **Data Protection Officer (DPO)**: Appoint a DPO or register with SDAIA. |
| | — **Cross-border data transfers**: Restricted unless to countries with adequate protection. Supabase self-hosted on Ubuntu in KSA avoids this. |
| | — **Privacy impact assessment**: Required before processing high-risk personal data. |
| **Gap in v1.0** | Privacy policy is marked "under legal review." Must be finalized by a Saudi-qualified lawyer before public launch. |
| **PDPL registration** | Register as a data controller with SDAIA/NDMO. |

### 2.4 Ministry of Commerce — Commercial Registration (Wathq)

**Authority:** Ministry of Commerce (وزارة التجارة)

| Item | Status / Action |
|------|-----------------|
| **Wathq API** | Provides API access to CR data (trade name, status, capital, owners, managers). |
| **Pilot stance (v1.0)** | Manual admin review of uploaded CR certificates. Wathq API deferred to post-pilot. |
| **Post-pilot** | Integrate Wathq API for automated CR verification. Display "Wathq Verified" badge. |

### 2.5 GEA — General Entertainment Authority

**Authority:** General Entertainment Authority (الهيئة العامة للترفيه)

| Item | Status / Action |
|------|-----------------|
| **Scope** | Regulates entertainment activities: live performances, events, festivals. |
| **Permit requirements** | Entertainment suppliers (performers, DJs, bands, entertainers) must hold a valid GEA permit. |
| **Platform obligation** | Marketplace must verify GEA permits for entertainment-category suppliers. |
| **Status in v1.0** | GEA permit is an uploadable document type. Admin manually reviews. Mentioned on landing page ("Approved by the General Entertainment Authority"). |
| **IMPORTANT** | Confirm with GEA whether Sevent needs its own platform-level approval, not just individual supplier permits. |

### 2.6 NCA — National Cybersecurity Authority

**Authority:** National Cybersecurity Authority (الهيئة الوطنية للأمن السيبراني)

| Item | Status / Action |
|------|-----------------|
| **ECC (Essential Cybersecurity Controls)** | Apply to critical national infrastructure. May not apply directly to a private marketplace, but compliance strengthens the platform. |
| **CSCC (Critical Systems Cybersecurity Controls)** | If Sevent is designated as critical infrastructure (unlikely at pilot stage). |
| **Practical measures** | |
| | — Encrypt data at rest (Supabase/Postgres encryption) |
| | — Encrypt data in transit (TLS 1.3 via Let's Encrypt on nginx) |
| | — Access control: RLS policies in Postgres, signed access tokens |
| | — Audit logging: Every state transition logs actor + timestamp ✅ Already implemented |
| | — Vulnerability management: Regular dependency scanning |
| | — Incident response plan: Document and maintain |
| | — Penetration testing: Conduct before launch |
| **Data residency** | Self-hosting on Ubuntu box in KSA ensures data stays in-Kingdom. ✅ |

### 2.7 Nafath — National Digital Identity

**Authority:** National Information Center (NIC) / Saudi Data & AI Authority (SDAIA)

| Item | Status / Action |
|------|-----------------|
| **What it is** | Biometric + national ID verification platform. |
| **Application** | Verify supplier identity by matching against national ID records. |
| **Status** | Deferred to post-pilot. Manual document review in v1.0. |

### 2.8 MCIT — E-Commerce & Digital Content Regulations

**Authority:** Ministry of Communications and Information Technology (وزارة الاتصالات وتقنية المعلومات)

| Item | Status / Action |
|------|-----------------|
| **E-Commerce Law** | Requires clear terms & conditions, privacy policy, return/cancellation policies, dispute resolution mechanism. |
| **Terms & Conditions** | Draft exists but marked "under legal review." Must be finalized. |
| **Privacy Policy** | Draft exists but marked "under legal review." Must be finalized. |
| **Commercial register display** | CR number must be displayed on the platform if Sevent is a registered company. |
| **Consumer protection** | Clear cancellation and refund policies, dispute resolution process. |

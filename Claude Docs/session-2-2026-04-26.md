### Specific defects fixed

**CRITICAL (security):**
- **SEC-1** Cross-tenant blob disclosure via `response_file_path` — RLS policy didn't constrain the column. Fixed by `quote_proposal_requests_guard_biu` trigger requiring `response_file_path LIKE supplier_id||'/proposal-responses/%'` + action-side `assertPathBelongsToSupplier`.
- **SEC-2** Status-transition hijack — supplier could flip fulfilled→pending. Fixed by trigger pinning `quote_id`/`requested_by`/`requested_at` immutable and only allowing `pending→fulfilled` (supplier path) or `pending→cancelled` (organizer path).
- **SEC-3** 0-rows-affected UPDATE leaked blob + lied "success" — supabase-js returns `error: null` with empty data on a CAS no-match. Fixed by `.select("id")` and length check; on 0 rows, remove the storage object and return a friendly error.

**CRITICAL (rendering):**
- **UI-1** Sticky `<th>` + `border-collapse` dropped borders during scroll on Chromium. Fixed by `border-separate border-spacing-0` + cell-level borders.
- **UI-2** Translucent `bg-muted/30` on sticky cells leaked supplier columns through. Fixed by opaque `bg-muted` on sticky `<th>`s.

**CRITICAL (perf):**
- **PERF-1** Sequential signed-URL minting per quote (15 round-trips for 5 suppliers). Fixed by `Promise.all(quotes.map(async q => ...))` + parallel inner `Promise.all([techPromise, logoPromise, fulfilledPromise])`.

**HIGH:**
- **SEC-4** Notifications inbox didn't render new kinds → fall-through default. Added cases in `iconForKind`, `linkForKind`, `kindKeys`; added i18n strings; modified `requestProposalAction` to look up + include `invite_id` in the payload.
- **SEC-5** Orphaned blobs — re-request-after-fulfilled or quote cascade-delete left storage entries. Added `quote_proposal_request_orphans` queue table + AFTER UPDATE/DELETE trigger that enqueues paths for a future cleanup job.
- **DATA-1** Unvalidated `snapshot_jsonb` cast → renderer crash on stale snapshot. Added `QuoteSnapshotSchema` (Zod, `.passthrough()`) + `parseQuoteSnapshot()` helper. All three boundaries now use it.
- **DATA-2** CSV download mid-stream redirected logged-out users to a saved `quotes-…csv` containing the sign-in HTML. Fixed by try/catch around the loader; on `NEXT_REDIRECT` digest, return 401 plain text.
- **DATA-3** VAT label was baked from the first supplier's rate (`{pct}` interpolated once). Misleading when rates differ. Fixed by switching the row label to non-interpolated `vatRow` and putting `${pct}%` inside each cell. Same fix in print page.
- **A11Y-1** Expand toggle missing `aria-expanded`/`aria-controls`. Added.
- **A11Y-2** `title=` tooltip on conflict badge isn't accessible. Replaced with `aria-label`.
- **UX-1** No "Clear filters" recovery from zero results. Added a button in the empty-after-filter state.
- **UX-2** Upload form lacked client-side size/type guard. Tightened.

**MEDIUM:**
- `requested_by` integrity coercion via trigger.
- Migration idempotency (`if not exists` everywhere).
- RfpCell silent cancel-success → transient success Alert.
- Print font race → `await document.fonts.ready` before `window.print()`.
- AutoPrint reprints on revisit → gated behind `?auto=1`.
- Print table-layout fixed + per-column widths.
- Invite-source non-determinism → `.order("sent_at", desc)` + `if (!has)` guard.
- CSV cell cap (32k Excel limit) → `capCell()` helper with item-boundary trim.
- `text-[10px]` Arabic badges → bumped to `text-[11px]`.
- Skipped quotes logged via `console.warn`; new `skipped: string[]` on the dataset.

## 6. Live testing — two follow-ups

User started step-by-step testing and surfaced two issues:

### 6a. Supplier inbox didn't surface pending RFP requests

**Symptom**: as supplier (test30), the `/supplier/rfqs` inbox showed "no open invites" because the invite was already in `quoted` status (moved to "Past activity"). The pending RFP request was buried — supplier had to expand past activity, find the right RFQ, and click in.

**Fix**: added an "Action required — proposal requests" section at the top of `/supplier/rfqs` (before open invites). Lists every pending `quote_proposal_request` for the supplier with category, city, event date, organizer message (or default body), relative timestamp, and an **Upload proposal** button that links straight to `/supplier/rfqs/{invite_id}/proposal-upload`. Header gets a second orange badge counting pending requests.

Files: `src/app/(supplier)/supplier/rfqs/page.tsx`; new keys `supplier.rfp.pending`, `inboxBadge` (ICU plural), `inboxSectionHeading`, `requestedRelative` in en + ar.

### 6b. Source label rendered raw i18n key on organizer RFQ detail

**Symptom**: the invites table on `/organizer/rfqs/{id}` showed `organizer.rfqs.source.self_applied` literal instead of the Arabic label.

**Cause**: `src/app/(organizer)/organizer/rfqs/[id]/page.tsx:306` called `t(\`source.\${inv.source}\`)` against a translator scoped to `organizer.rfqs`, but `source.*` keys don't exist there. They live at `organizer.quote.sourceBadge.*` (added during the comparison work).

**Fix**: bound a second translator `tSource = await getTranslations("organizer.quote.sourceBadge")` and switched the cell to `tSource(inv.source)` with a narrow check on the three known source values.

Files: `src/app/(organizer)/organizer/rfqs/[id]/page.tsx`.
